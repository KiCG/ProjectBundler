# Project Bundler
Project Bundler は、Blenderのプロジェクトファイルと、そこで使用されているテクスチャなどの外部リソースを一括で整理・保存するためのアドオンです。

プロジェクトを別名保存する際、自動的に専用フォルダを作成し、その中に .blend ファイルと textures フォルダを生成してリソースを格納します。データの受け渡しやバックアップ、アーカイブ化に最適です。

## ✨ 特徴
フォルダごとの自動書き出し: 指定した場所に プロジェクト名_Bundle というフォルダを自動生成します。

リソースの自動収集: 使用している画像テクスチャなどを自動的に収集し、相対パスでリンクし直します。

階層の整理: テクスチャ類は textures フォルダにまとめられるため、フォルダ内が散らかりません。

エラー回避: リンク切れのファイルがある場合でも処理を中断せず、存在するファイルのみをパックして書き出します（欠落ファイルはスキップ）。

自動オープン: 書き出し完了後、自動的に保存先のフォルダを開きます（Windows / Mac対応）。

## 📂 出力イメージ
実行すると、以下のようなフォルダ構成で保存されます。

Plaintext
MyCharacter_Bundle/        <-- 自動生成される親フォルダ
  ├── MyCharacter.blend    <-- プロジェクトファイル
  └── textures/            <-- リソースフォルダ
       ├── base_color.png
       ├── normal_map.png
       └── background.jpg
## 📥 インストール方法
このページから Project_Bundler.py (またはzipファイル) をダウンロードします。

Blenderを起動し、メニューから 編集 (Edit) > プリファレンス (Preferences) を開きます。

アドオン (Add-ons) タブを選択し、インストール (Install...) ボタンをクリックします。

ダウンロードしたファイルを選択してインストールします。

リストに表示された 「Import-Export: Project Bundler」 のチェックボックスをオンにして有効化します。

## 🚀 使い方
書き出したいBlenderプロジェクトを開きます。

上部メニューの ファイル (File) > エクスポート (Export) から 「Project Bundler (Folder)」 を選択します。

ファイルブラウザが表示されるので、保存したい場所を選択して「Export Project Bundle」ボタンを押します。

処理が完了すると、生成されたフォルダが自動的に開きます。

## ⚠️ 注意事項
リンク切れファイルについて: 元のプロジェクトで「ピンク色（missing）」になっているテクスチャは、書き出し時にスキップされます。必要な場合は、事前に「外部データ > 欠落ファイルを検索」でリンクを修復してください。

上書きについて: 同名のフォルダが既に存在する場合、ファイルは上書き・統合されます。

## 💻 動作環境
Blender 4.0 以上推奨

OS: Windows / macOS / Linux
